GitHub的使用

2016-06-04
  • 工具

第一步

- mkdir  + <i> filenme </i>

新建一个文件夹

- cd  + <i>filename</i>

进入文件夹

- git init

初始化操仓库（会生成一个隐藏的git文件夹）

- touch + <i>filename</i>

建立一个文件

第二步

- git add  + <i>filename</i>

将文件加入提交缓存空间

- git rm  --cached

将提交缓存空间里的文件清除

- git commit -m + <i>'修改注释'</i>

将缓存空间里的文件提交到仓库，修改注释用引号

- git log

查看提交记录

- git status

查看当前git目录的状态信息

第三步

- git branch

查看当前分支状态（有什么分支，和当前在哪个分支工作）

- git branch + <i>branchname</i>

新建一个分支（但是不会把工作对象切换到这个分支）

- git checkout + <i>branchname</i>

将工作对象切换到这个分支

- git chekout -b + <i>branchname</i>

新建一个分支，并将工作对象切换到这个分支

第四步

- git merge + <i>branchname</i>

将分支合并到master分支上来，前提是你要先把工作对象切换到master分支

- git -d + <i>branchname</i>

删除这个分支（前提是该分支已经合并到master分支）

- git -D + <i>branchname</i>

强制删除这个分支

第五步

- git tag

查看标签状态

- git tag + <i>tagname</i>

为代码添加标签

- git checkout +<i>tagname</i>

将代码切换到添加该标签时的状态

生成公钥

安装git是会自动生成公钥，在c/document and setting/user/username/.ssh里

- ssh-keygen -t rsa

指定 rsa 算法生成密钥，接着连续三个回车键（不需要输入密码），然后就会生成两个文件 id_rsa 和 id_rsa.pub ，而 id_rsa 是密钥，id_rsa.pub 就是公钥。这两文件默认分别在如下目录里生成：

Linux/Mac 系统 在 ~/.ssh 下，win系统在 /c/Documents and Settings/username/.ssh 下，都是隐藏文件.把 id_rsa.pub 的内容添加到 GitHub 上，这样你本地的 id_rsa 密钥跟 GitHub 上的 id_rsa.pub 公钥进行配对，授权成功才可以提交代码。

查看公钥

- ssh

查看公钥

第六步

- git pull origin master

向GItHub仓库提交文件

- git push origin master

从GitHub更新自己的文件

提交文件的两种方法

- git clone + <i>address</i>然后push

先克隆，再提交

- git remote +<i>remotename</i> + <i>address</i>然后push

先关联再提交

- git remote -v

查看当前的远程仓库
